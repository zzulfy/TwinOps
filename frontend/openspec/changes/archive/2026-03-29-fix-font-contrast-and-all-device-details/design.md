## Context

当前系统在白色主题下仍存在文本层级不清晰的问题，尤其在信息密集的面板区域会降低监控效率。与此同时，设备详情页在部分导航路径中仍会回退到单设备展示，与“默认查看全部设备详情”的目标不一致。该变更涉及设计令牌、详情页布局与路由容错三个模块，需要在实现前明确统一决策，避免再次出现行为回退。

## Goals / Non-Goals

**Goals:**
- 在白色主题下提升主文本与次级文本的对比度一致性，保证关键信息清晰可读。
- 确保设备详情页默认展示全部设备，不再依赖单设备上下文才能正确渲染。
- 使设备详情路由在无参数访问时仍稳定渲染全量设备视图。

**Non-Goals:**
- 不引入新的后端接口或数据模型。
- 不重构三维场景或设备标签渲染逻辑。
- 不调整与本次问题无关的组件视觉风格。

## Decisions

1. 采用“语义令牌收紧”策略而非组件内局部覆盖：统一修改 `src/assets/design-tokens.css` 的文本令牌，避免不同组件自行定义文本颜色导致回归。
2. 采用“全量设备聚合视图优先”策略：`DeviceDetailPage.vue` 直接消费设备集合并使用列表/网格渲染，单设备上下文仅作为可选过滤能力，而非默认入口。
3. 采用“路由无参数可用”策略：`/devices` 必须可独立渲染全量详情，避免依赖 query 参数导致页面退化。

## Risks / Trade-offs

- [Risk] 对比度提升可能改变现有视觉层级习惯 → Mitigation: 保持主次文本两级对比差，避免全部文本同等强调。
- [Risk] 全量设备卡片数量增加导致页面滚动变长 → Mitigation: 使用响应式网格与卡片尺寸约束控制阅读负担。
- [Risk] 路由兼容旧链接时出现行为分歧 → Mitigation: 旧参数路径保留可读，但无参数路径定义为稳定主路径。
