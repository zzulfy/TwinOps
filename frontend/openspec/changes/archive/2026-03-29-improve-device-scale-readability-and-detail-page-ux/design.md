## Context

当前问题同时涉及视觉可读性与信息完整性：设备规模模块字体过淡导致识别困难，设备详情区域仍带有黑色窗口风格且视觉粗糙，与白色主题不一致；此外详情容器在长内容场景下滚动受限，造成设备信息显示不全。该变更跨越 design tokens、详情面板组件与页面容器布局，需要统一方案避免局部修补反复回归。

## Goals / Non-Goals

**Goals:**
- 提升设备规模模块及相关高密度文本的对比度，确保白底下清晰可读。
- 将设备详情面板改为更美观、统一的白色主题风格，提升信息层次与可扫读性。
- 保障详情页面和面板容器可滚动，支持完整展示全量设备信息。
- 消除因容器高度/溢出策略导致的设备信息遗漏。

**Non-Goals:**
- 不改动后端接口与设备数据结构。
- 不重写 3D 场景和标签拾取逻辑。
- 不引入新的 UI 框架或第三方视觉库。

## Decisions

1. 采用“令牌优先”方式修复可读性：通过 `src/assets/design-tokens.css` 统一提升文本对比度，避免组件局部硬编码颜色再次变浅。
2. 将 `DeviceDetailPanel` 从黑色窗口视觉迁移到白色主题语义面板：使用更轻边框、分层背景与清晰标题层级。
3. 页面采用“外层页面滚动 + 面板内容自适应”策略：`DeviceDetailPage` 保证纵向滚动可用，设备卡片网格允许自然换行并避免裁切。
4. 明确“信息完整优先”：当设备数量增多时优先保留完整可滚动展示，不以固定高度截断详情字段。

## Risks / Trade-offs

- [Risk] 提高对比度后页面视觉可能显得“更重” → Mitigation: 保留主/次级文本层级差，避免全部文本同强度。
- [Risk] 面板白化后与部分旧深色子样式冲突 → Mitigation: 清理 `DeviceDetailPanel` 内残留深色变量，统一改为 token 驱动。
- [Risk] 开放滚动后可能影响首屏卡片密度 → Mitigation: 通过响应式网格和合理间距维持浏览效率。
