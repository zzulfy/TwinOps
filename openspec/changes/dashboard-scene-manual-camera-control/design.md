## Context

Dashboard 设备仿真场景当前仍包含自动旋转/浮动动画，用户在定位设备细节时会被持续位移打断。同时相机控制边界偏紧，拖动视角后无法完整查看全景。该变更需要将场景控制权从系统自动运动切换为用户手动操作，并保持现有渲染性能与稳定性。

## Goals / Non-Goals

**Goals:**
- 禁用设备仿真场景的自动移动行为，避免画面自行旋转或上下漂移。
- 保留鼠标视角交互并提升全景可达范围（旋转/缩放/可控平移）。
- 在不重构渲染架构的前提下完成行为修复并可回归验证。

**Non-Goals:**
- 不更换 Three.js 或 OrbitControls 技术栈。
- 不新增业务巡检路径、自动演示相机轨迹等功能。
- 不调整看板其它业务组件（告警、图表、设备列表）逻辑。

## Decisions

1. **从动画循环中移除自动位移**
   - 删除场景对象每帧 `rotation`/`position` 自动更新逻辑，仅保留用户交互驱动的视角变化。
   - 备选方案是降低自动速度，但仍会影响用户观察，故拒绝。

2. **强化 OrbitControls 手动控制能力**
   - 启用旋转与缩放，允许适度平移，放宽极角和距离范围，确保可查看全景。
   - 备选方案是保留禁用平移，仅靠旋转。实际容易出现局部遮挡无法绕过，故采用可控平移。

3. **保持默认视角与性能稳定**
   - 维持当前相机初始位置和 deferred loading，仅修改控制参数和自动运动逻辑。
   - 这样风险最小，不引入新的加载链路或渲染瓶颈。

## Risks / Trade-offs

- **[Risk] 放宽控制后用户可能将镜头拖到不理想角度** → **Mitigation**: 设置合理的 `min/maxDistance` 与 `maxPolarAngle` 约束。  
- **[Risk] 去除自动运动后场景“动感”降低** → **Mitigation**: 优先保证可用性和可观察性，动效不再作为主目标。  
- **[Trade-off] 启用平移会增加交互复杂度** → **Mitigation**: 保持阻尼与边界限制，避免失控漂移。
