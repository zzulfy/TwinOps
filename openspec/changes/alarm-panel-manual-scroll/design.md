## Context

预警面板目前仍包含自动滚动行为，虽有性能优化，但用户体验诉求已明确转向“手动滑动查看”。在告警信息密集场景下，自动位移会打断阅读节奏，并导致用户在定位某条告警时需要反复等待轮播周期。

## Goals / Non-Goals

**Goals:**
- 将预警列表交互改为手动滚动为主，不再自动轮播。
- 保持告警筛选、数据刷新、状态展示逻辑不变。
- 在无自动滚动前提下维持列表渲染稳定，无跳动与焦点丢失。

**Non-Goals:**
- 不修改后端告警 API 契约与数据结构。
- 不新增复杂交互组件，仅使用浏览器原生滚动能力。
- 不重构其他面板滚动行为。

## Decisions

- **Decision 1: 移除自动轮播定时器与动画状态机**  
  删除自动滚动相关 interval/animation 控制，避免持续动画占用与被动位移。

- **Decision 2: 使用可滚动容器承载列表**  
  通过固定可视区 + `overflow-y: auto` 支持用户手动滚动，保障滚动行为可预测。

- **Decision 3: 保留自动刷新但不触发滚动定位变化**  
  告警数据可继续按既有刷新策略更新，但不主动改动滚动位置，避免刷新时“跳读”。

- **Decision 4: 回归脚本从“自动轮播验证”改为“无自动轮播验证”**  
  更新 smoke 断言：验证列表不会自动位移，且用户可手动滚动浏览数据。

## Risks / Trade-offs

- **[Risk]** 去掉自动轮播后首屏告警覆盖率下降 → **Mitigation**: 保证滚动容器可顺畅手动浏览并保留筛选标签。
- **[Risk]** 数据刷新时列表高度变化影响滚动体验 → **Mitigation**: 控制 item 高度与容器布局稳定，避免刷新后突变。
- **[Trade-off]** 从“动态展示”转向“用户主动操作” → **Mitigation**: 以可读性和可控性优先，符合用户明确反馈。
