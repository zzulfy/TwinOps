## Context

当前预警情况面板（`WidgetPanel06.vue`）滚动实现采用定时器 + `top` 动画 + 列表 `shift/push` 重排组合，在自动刷新与交互并存时容易产生掉帧和抖动。问题集中在布局抖动、动画重入、列表重排成本以及样式过度过宽（如 `transition: all`），导致告警阅读体验不稳定。

## Goals / Non-Goals

**Goals:**
- 预警列表滚动在常规设备环境下保持连续平滑，无明显卡顿。
- 轮转逻辑与动画节奏一致，避免滚动中重复触发引发抖动。
- 降低滚动动画期间的布局与重绘开销。
- 为滚动优化建立可重复回归验证（功能正确性 + 性能感知）。

**Non-Goals:**
- 不改变告警业务数据来源与状态语义。
- 不引入额外重型依赖（如新的动画库）。
- 不改造整个 Dashboard 动画体系，仅聚焦预警列表滚动链路。

## Decisions

- **Decision 1: 使用 transform 驱动滚动位移，替换 top 动画**
  通过 `transform: translateY(...)` 做视觉位移，减少布局重排，提升动画流畅度。

- **Decision 2: 引入单次滚动状态锁，防止动画重入**
  增加 `isAnimating` 状态，保证一次滚动周期完成前不会再次触发轮转。

- **Decision 3: 滚动完成后再执行队列轮转**
  先完成视觉动画，再进行最小化数据轮转，避免动画过程中数据结构变化带来的闪烁。

- **Decision 4: 收敛样式过渡范围**
  避免 `transition: all`，只保留必要属性过渡，降低每帧样式计算开销。

- **Decision 5: 增加性能与行为回归脚本**
  通过 smoke/regression 脚本覆盖滚动触发、轮转正确性和视觉连续性，防止后续回归。

## Risks / Trade-offs

- **[Risk]** transform 动画时机与数据轮转时机不一致 → **Mitigation**: 统一以动画结束事件或固定周期结尾点执行轮转。
- **[Risk]** 状态锁处理不当导致“停转” → **Mitigation**: 在异常分支也确保解锁，并增加测试覆盖。
- **[Trade-off]** 更保守的滚动节奏可能降低“动效强度” → **Mitigation**: 以稳定可读优先，必要时开放节奏参数调优。
